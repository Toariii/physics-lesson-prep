# Course Planning

## Planning Entry Gate

Enter S7 only after the S6 Evidence Package receives teacher option A, the primary textbook or knowledge mainline is confirmed, and there are no blocking conflicts. The Course Requirements Confirmation Sheet remains current, including goals, deadlines, lesson conditions, homework capacity, equipment, and required outputs. If any condition changes, classify its impact and roll back to the earliest affected state rather than planning from stale assumptions.

## Teaching Capacity Calculation

Record and calculate these fields explicitly:

- Calendar weeks and planned weeks available.
- Planned teaching weeks after excluding calendar weeks that contain no teaching.
- Lessons per week and Minutes per lesson.
- Theoretical lessons before deductions.
- Holidays/cancellations.
- Diagnostic lessons allocation.
- Module/stage assessment lessons allocation.
- Module/stage assessments summary across the cycle.
- Review/revision/simulation allocation.
- Effective lesson-equivalents/adjustments for shortened or extended sessions.
- A 5-10% contingency reserve, recorded as contingency percent and lesson equivalents.
- Usable teaching lessons after every deduction.

Use equivalent lessons as the unit throughout. Theoretical lesson-equivalents = planned teaching weeks * lessons per week. Convert minutes to lesson equivalents using Minutes per lesson as the denominator before combining values. Define the signed effective-session adjustment as positive for extended sessions or equivalent added teaching and negative for shortened sessions or lost partial sessions. Then calculate: usable lesson equivalents = theoretical lesson-equivalents + effective-session adjustment - cancellations - diagnostic lessons - module/stage assessment lessons - revision/simulation - contingency lesson-equivalents.

Calculate the contingency percentage from theoretical lesson-equivalents after known fixed cancellations, before other planned deductions. Record both that base and the resulting contingency percent and lesson equivalents; do not recalculate contingency from an already reduced usable total. Keep every cancellation, diagnostic lesson, module/stage assessment lesson, revision/simulation lesson, contingency lesson-equivalent, and signed adjustment in exactly one field to avoid double counting. Do not count homework as lesson capacity. Homework is a separate learner-load assumption and may support, but never replace, scheduled teaching. For variable attendance, show a range and a conservative case. State every assumption, including partial weeks, shortened lessons, known events, expected absence, conversion basis, and rounding. Recalculate when an assumption changes.

## Goal-Capacity Conflict

When goals exceed usable capacity, do not fabricate capacity or silently compress learning. Present all three routes with lesson allocation and consequences:

- Minimum viable route: protect core outcomes and defer lower-priority content.
- Recommended route: preserve understanding, practice, and feedback.
- Enhanced route: requires more lesson capacity, homework capacity, or time.

For each route, show the impact on goals, assessment readiness, deferred content, and risks. The teacher selects the route; the system does not choose by default. A minimum viable route states the core outcomes retained and exactly what is deferred. A recommended route includes enough explanation, practice, checking, and feedback to support the confirmed outcomes. An enhanced route identifies the additional lessons, homework, or calendar extension required.

Record a Selected capacity route and its basis. When a capacity conflict exists, the teacher must explicitly select Minimum viable, Recommended, or Enhanced before confirmation; Option A is unavailable while a capacity conflict has no selected route. When no conflict exists, record either Recommended or `No conflict - confirmed framework` as the selected/default route and state the capacity basis. Carry the selected route and its implications into S8, including retained outcomes, deferred work, assessment consequences, added demands, and accepted risks.

## Course-Cycle Layers

Build the full course-cycle framework at these connected layers:

1. Define final measurable outcomes.
2. Allocate stages: entrance diagnosis and prerequisite repair if needed; concepts and models; basic application and representations; integrated work and method selection; exam, transfer, or competition work; summary and repair.
3. Sequence modules and show the dependency graph.
4. Specify minimum prerequisite remediation, not broad repetition.
5. Set the concept/practice proportion by stage.
6. State module exit criteria.
7. Place assessment nodes.
8. Reserve review and contingency.
9. Record risks, triggers, and alternative routes.

Every layer must remain traceable to confirmed course evidence, learner evidence, the selected goal strategy, and usable teaching capacity.

## Module Dependencies And Minimum Remediation

Express dependencies as observable prerequisites, for example: vectors and algebraic rearrangement -> force decomposition -> Newton-law modeling; graph slope and area -> kinematics representations -> multi-stage motion; energy definitions -> system choice -> conservation modeling. A dependency is included only when failure would block the next module or materially distort its evidence.

Use minimum prerequisite remediation: repair only the missing concept, mathematics operation, representation, notation, or method needed for the next module. This is not a reteach of the entire prior course. State the trigger, short remediation task, evidence check, and the effect on allocation. If the gap is larger than the reserve can absorb, return to the goal-capacity routes for teacher choice.

## Module Exit Criteria

Each module defines evidence-based exit criteria across knowledge, skill, notation, transfer, and accuracy/time appropriate to the goal and learner stage. Identify the evidence source, such as observed explanation, worked solution, independent item set, practical record, quiz, or timed performance.

If criteria are not met, provide a bounded remediation route, its capacity cost, and a teacher choice: remediate now, revise the target, adjust allocation, or accept a documented risk. The learner does not advance automatically because planned time elapsed.

## Assessment Nodes

Use assessment nodes with an explicit purpose, evidence, and response:

- Entrance diagnosis: establish prerequisite and current-course starting evidence; respond with targeted remediation or confirmation of the starting module.
- In-lesson formative check: test the current representation, reasoning step, or misconception; respond immediately with explanation, example change, or practice adjustment.
- Module quiz: evaluate exit criteria; respond with remediation, allocation change, or teacher-approved progression.
- Stage assessment: test integration across modules and method selection; respond by revising the next stage and review reserve.
- Comprehensive assessment: test cumulative retention and transfer; respond with priority repair and route-risk updates.
- Mock/final output: rehearse the target format, timing, marking, or authentic performance; respond with final retraining and a readiness judgment.

For preview goals, adapt assessment evidence toward concept maps and explanation rather than premature exam imitation. For competition goals, adapt it toward strategy and nonstandard problem solving, multiple routes, proof quality, and justified approximation.

## Four Goal Strategies

- For score improvement: score-loss analysis -> high-frequency repair -> item and marking analysis -> timed mixed practice -> simulation -> retraining. Tie repair frequency to actual errors and official marking demands.
- For synchronized consolidation: synchronized pre-activation -> concept repair -> schoolwork correction -> transfer -> assessment. Keep sequence aligned with school progress while correcting misconceptions rather than merely repeating classwork.
- For advance preparation: advance familiarity -> understanding -> readiness -> cognitive-load reduction -> mathematics preparation. The objective is readiness and lower future cognitive load, not assigning all hard problems early.
- For competition or enrichment: competition mathematics route -> model library -> multiple methods -> approximation -> dimensional analysis -> proof -> nonstandard problems -> strategy -> official problems. Use official problems to calibrate route choice, communication, and strategic flexibility.

## Concept-Practice Allocation

For mixed needs, state explicit proportions or stage-by-stage changes. Distinguish explanation, guided practice, independent practice, and review; do not hide any of them under a single practice percentage. Explain how the allocation serves the selected goal strategy and learner evidence. If the requested proportion conflicts with the goal, assessment format, usable capacity, or exit criteria, show the conflict and offer revised proportions for teacher selection.

## Risks And Alternative Routes

Record each risk with a trigger and an alternative route, plus the state rollback when relevant:

- Insufficient time: trigger when usable capacity drops or milestones slip; reduce to the selected lower route or return to S7 for allocation confirmation.
- Math gap: trigger on repeated prerequisite failure; apply minimum remediation or return to S3/S4 if new diagnostic evidence is required.
- School progress change: trigger when the taught sequence or assessment date moves; resequence in S7 and revisit S2 if goals change.
- Homework noncompletion: trigger on repeated missing independent evidence; move essential practice into lessons and recalculate capacity.
- Source mismatch: trigger when materials conflict with the confirmed course/version; return to the earliest relevant state from S1-S6.
- Score fluctuation: trigger on repeated variance beyond the stated confidence band; add diagnostic evidence, retraining, or a revised readiness claim.
- Equipment constraint: trigger when required apparatus or access is unavailable; use an evidence-equivalent alternative or return to S4.
- Goal change: trigger on a teacher-confirmed new priority or success criterion; return to S2 and then rebuild S7 as required.

## Rolling Two-To-Four-Lesson Rule

At S7, confirm the full course-cycle framework only. Prepare the next two to four lessons after confirmation: the first lesson is fully fixed, while lessons 2-4 remain adjustable using new evidence. Do not detail all future lessons, because later content must respond to learner performance, schedule changes, and assessment evidence.

If a two-to-four-lesson batch has already been delivered, wait in S8 without regenerating it. S8 remains active through the current two-to-four-lesson batch. Lesson-level observations may be logged during S8 and can trigger urgent rollback to the earliest affected state, but they do not normally start S9. S9 begins only when the teacher reports the current batch taught or completed; then perform batch reflection and record updates before fixing the next first lesson.

## Course-Cycle Confirmation

Offer these exact teacher options:

- A - confirm and advance to S8.
- B - change proportions and stay in S7.
- C - change order/allocation and stay in S7.
- D - change objectives and return to S2 or S7 according to impact.
- E - add evidence and return to the earliest affected state from S1-S6.

Only option A advances to S8. Option A also requires the Selected capacity route and basis described above; if a capacity conflict exists, the explicit teacher-selected route is mandatory. Record the selected option, carry the selected route and its implications into S8, and record any resulting rollback before producing lesson materials.
