# Material Packages

## S8 Entry Gate And Common Header

S8 may begin only with the confirmed Course Evidence Package, confirmed primary textbook or knowledge mainline, selected capacity route, confirmed course-cycle framework, and no blocking unresolved item. Carry the selected capacity route and its implications into every batch. Match the explicit deliverables confirmed in S4 and S7; do not silently substitute a worksheet, slide deck, experiment, or answer format.

Every batch uses a common header recording the anonymous course record, current state, course/version, primary textbook and chapters, course-cycle stage, learner evidence used, package mode, lessons in the batch, source status, release label, and teacher checks still required.

Write the common header in Chinese for teachers. Keep course codes, book titles, formulas, source titles, and student-facing English problem prompts in their original language. Do not show `Current state: S8`; use `当前步骤: 准备本次课程`.

## Concept-Focused Package

A concept-focused package contains all of the following:

- objectives and observable exit criteria;
- prerequisites and a central question;
- a phenomenon or context that gives the concept a physical purpose;
- an accurate concept statement;
- the model, assumptions, boundaries, and approximation regime;
- derivation depth aligned with the confirmed course and primary textbook;
- multiple representations, such as verbal, diagrammatic, graphical, algebraic, numerical, or experimental;
- a real-life or engineering example that names the real system, states the idealization, identifies what the model explains, and identifies what it does not explain;
- counterexamples and easy-confusion cases;
- a teacher question chain;
- a concept check, guided example, and independent foundation practice;
- answers and diagnostic interpretations that connect representative errors to likely obstacles;
- a concise summary.

If the model must be established first, do not start with an equation. Introduce the system, interaction, assumptions, and physical relationship before symbolic manipulation. A daily-life example is not automatically correct: distinguish the observed real process from the chosen idealization, avoid false causal stories, and state the limits of the analogy.

## Practice-Focused Package

A practice-focused package contains a learning goal; a question-type map; a selection rationale; source and provenance; official, licensed, original, or adapted status; difficulty; cognitive demand; expected time; a clean student version; a separate teacher solution; marking points; each common error and its cause; a meaningful variant; a second-attempt problem; and an error-record field.

Classify cognitive demand using the exact applicable labels: concept identification, model selection, equation construction, representation conversion, multistep reasoning, evaluation, and non-standard transfer. Do not use a number-only change as a variant. Change a representation, condition, model choice, reasoning demand, or transfer context. Fully and independently solve every adapted problem rather than assuming the source solution survives the adaptation.

## Mixed Package

Use this exact instructional sequence:

`diagnosis -> targeted explanation -> teacher modeling -> scaffolded practice -> independent practice -> error discussion -> variant transfer -> exit evidence`

The targeted explanation addresses the diagnosed obstacle, not the whole chapter. Each transition uses learner evidence, and the exit evidence determines the later branch.

## Competition Or Enrichment Package

A competition or enrichment package contains context with minimal information, model construction, required mathematics, graded hints, multiple solutions where valid, method comparison, the breakthrough point, plausible dead ends, generalization, strategy reflection, and a complete solution. Stage hints from orientation, to model cue, to mathematical cue, to a near-complete scaffold; do not reveal the decisive step in the first hint. Use official problem scope and current rules where the activity is tied to an identifiable competition.

## Student And Teacher Version Separation

Produce a clean student version and a separate teacher version. The student version contains no hidden commentary, answers, marking guidance, anticipated-error notes, or diagnostic interpretation. Keep answers, marking points, anticipated errors and responses, branch guidance, and diagnostic interpretations in the teacher version. Add accessibility, language support, notation glossaries, visual alternatives, or accommodation notes when relevant to the confirmed learner and setting.

If the teacher has not confirmed bilingual teaching, default explanations, headers, instructions, and reply prompts to Chinese. Preserve English only where it is the learning target, such as English problem stems, vocabulary, formulas, course codes, source names, or textbook names.

## Question Provenance

For every question record title or source, year/series/edition and item when applicable, official/licensed/original/adapted status, author or institution, URL or bibliographic record where available, license/reuse status, intended use, and verification status. Never label an unknown source as official or as a past-paper question. Respect copyright and quotation limits; prefer lawful links, bibliographic references, licensed use, or genuinely original work.

Original and adapted questions require an independent physics audit and independent solution. An adapted question must identify what changed and must be fully re-solved, including diagram, values, answer, marking points, and acceptable alternatives.

## Standard Solution Contract

Use these exact solution lines, in this order:

```text
Model and known conditions
Symbols, frame, and sign convention
Physical principle
Derivation or calculation
Final answer
Units and significant figures
Physical interpretation
Independent check
Marking points
Accepted alternative methods
```

Where more than one result is defensible, state the alternative acceptable range, tolerance, equivalent form, or method and explain how it earns marks.

## Lesson One Versus Lessons Two To Four

Lesson 1 is fully fixed for delivery. Lessons 2-4 are adjustable branches driven by lesson-level evidence; do not create a fully detailed future course. Preserve the confirmed framework, selected capacity route, evidence basis, dependencies, and exit criteria. If an earlier lesson changes the diagnosis or pace, revise the affected later draft and record the reason; do not silently regenerate the whole batch.

## Teacher Decision And Export Options

End each prepared lesson or batch with Chinese teacher decisions:

```text
教师决定: 批准第1节课 / 修改第1节课 / 暂不使用
导出选项: 生成PDF / 生成PPT / PDF和PPT都生成 / 暂不导出
```

If the teacher chooses an export, require an output path and read `reflection-and-records.md` before writing. Default exported files exclude real names and use the anonymous course ID or generic learner label. If the teacher also uses a local named roster, offer a separate roster update after the lesson is taught.

## Teacher-Review Label

Use the release label `teacher-review-ready` only when the course gate, education-design gate, physics audit, intended-use gate, and all source, diagram, solution, feasibility, safety, accessibility, and version-separation checks pass. Otherwise use `draft` and list the exact blockers or unresolved checks. Even a `teacher-review-ready` package still requires teacher review for local fit, practical safety, school policy, language, accessibility, and accommodation.
